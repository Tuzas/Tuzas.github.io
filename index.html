Aviation Accidents

Radvilas Mikonis 5/18/2020

Aviation is a relatively recent advancement in human history, however, it has become one of the most important ways people travel and connect across the world. This reliance on aviation has brought with it the requirement for high reliability and safety. However, accidents still occur and are carefully documented in order to explore the factors that contributed to these events. As time has went on, new regulations and improvements in technology should have made air travel safer for all. But is this a case? We can use the NTSB aviation accident information to see the locations, accidents, and the extent of injuries and casualties from these events. We should make the distinction that since we are not monitoring flying as whole, we cannot make meaningful inferences into the reliability of individual aircraft, so we will look at the extent of injuries and fatalities over time to see if the chances of surviving an aviation accident have increased, and if so, to what extent the make of the aircraft influences this.

Data¶

First we will want to gather our data that we will be able to analyze and hopefully answer our questions. There are two methods to obtain data, one way is to scrape data directly from websites and then organize it into a database by yourself. The other option is to find a file that contains the information for us.

We will opt for the second option as the data is already available so it would be easier to make use of existing data rather than try to create our own database.

So, since aviation data is something that is fairly important to aviation regulatory and safety bodies, we can find a dataset at the NTSB site.

They provide the dataset in a file called "AviationData.txt". We can download the file and open it up locally, noting that the delimiter being used is the "|" character. Knowing this we provide the path to the text file and the delimeter, we can now open the data set in python:

It is important to look over the dataset and make a note of how complete it is, and the significance of the values provided.

Right away we notice a problem. It appears that our dataset is missing values in some areas, and that some of the data is not properly formatted, which will make analysis more difficult going forward if we don't correct these issues.

For example, we have some columns that provide the numbers for the amount of people that were injured or fatally wounded. In some cases, we see 0's in some columns, while in other cases we see absent values. Does this mean there were also 0 people injured or do we just not know?

Before we can proceed we will have to make some decisions as to how we want to tidy the data including:

  • Which columns do we want to remove?
  • What columns do we want to add?
  • What columns do we want to consolidate?
  • How will we treat null values?
  • Is there a way to gracefully resolve the null values for some of these columns?

Data Tidying¶

Some columns offer little in the way of analysis or offer unneccessary information, so we can trim them. This allows us to focus on the remaining columns without having to worry about any extra information. These columns include:

  • Event Id
  • Accident Number
  • Airport Code
  • Airport Name
  • Registration Number
  • Number of Engines
  • Engine Type
  • FAR Description
  • Schedule
  • Air Carrier
  • Report Status
  • Publication Date
  • A hidden column with no title

Next, we want to figure out how we want to treat the null values of the injury columns. We can look at the NTSB website and see the description for all fields. However, it seems they give no direct information that could help us.

Instead, we can look up the most recent event, the 05/09/2020 accident at Haskell, OK and see that the news reported only one person who was flying, the pilot, and that they suffered injuries. So we can surmise that the one person listed is the pilot and all the null values should be 0's as a result. This doesn't apply to any columns where the injury severity is unavailable. So, we will remove the columns that are list injury severity as unavailable before proceeding and then convert them to integers.

Preparing Data for Analysis¶

Next we will want to decide what kind of dataframe we want to create from the original. We have many options here, including exploring the extent of injuries, the effect of weather conditions on plane crashes, the probability of surviving a plane crash when the aircraft damage is high, or looking at the effects on fatalities when the purpose of flight is personal rather than non-personal. Really, any aspect of the above data set can be explored.

In our case, we can keep it simple and opt to see which major airplane manufacturers have the highest survival rates when involved in a crash. To do this we will have to create a new dataframe and remove any columns we aren't interested in analyzing right now.

We get some interesting initial results.

There seems to be a propensity for large jet manufacturers such as boeing and airbus to have survivors when their planes crash, with the values being fairly low and consistent over time. This makes sense since these companies rely on good safety measures to maintain good rapport with the public.

This is in contrast to smaller, more personal plane manufacturers such as cessna and beech, which had an above average mortality rate which has remained fairly stable, pointing to the idea that mortality rates are not decreasing and that personal small aircraft seem to be more dangerous than commercial flying.

Interesting datapoints exist, such as in the case of "Robinson", where we see a fairly stable distribution for the first 4 points, followed by a large rise. Further research shows that Robinson helicopters seem to have had increased susceptibility to mast bumping incidents from 2006 to 2016, matching up with the spike in fatal crashes and thus decreased survivability of the passengers and crew. It seems the issue was big enough that there was a lawsuit brought against them after a helicopter suffered an in-flight breakup following mast bumping.

However, we can do further analysis to get a better interpretation of our major results.

Looking at our regression summary, we can see that our adjusted R-squared value isn't very high, showing that our model doesn't fit extremely well. However, the f-statistic shows us that there is a good amount of linear relationship between the target variable and feature variables.

We can see the general trends between the maker and the chance of a fatality in a crash, with boeiing and bombardier being fairly low while beech seems to be the highest. As for the year ranges, we can see that they are not decreasing over time, meaning that if the data model is accurate, then it's not necessarily the case that survival chances have gone up of living through a plane crash since the 70's.

As for the accuracy and representation of the model, it's obvious there are other factors that affect fatality rates, but the makes aren't completely inconsequential.

Conclusion¶

In conclusion we can't say that the chances of surviving a crash has increased over time in for all makes based on the analysis we did. Some considerations would be to take the fatality/passenger ratio per crash and evaluate that rather than splitting it up by make, and seeing what trends appear and whether there is an overall increase in survival rates. Furthermore, there are other variables that must be responsible for the fatalities, date and make don't seem to be the strictest of factors in this analysis, yet they did have some effect. We could also see some manufacturers were fairly consistent while others had periodic problems that cannot be easily predicted by this model. So in that sense, some makes are erratic while others are fairly stable in their manufacturing, but there doesn't seem to be a good way to predict when a make will suffer issues.
